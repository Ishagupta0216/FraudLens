# FraudLens Backend

FraudLens is a Flask-based fraud detection backend for analyzing UPI transaction narrations and SMS messages. It uses TF-IDF features with a Logistic Regression classifier, then combines the model score with keyword-based rules for category detection, suspicious keyword extraction, risk levels, and analyst-style explanations.

## Detection Engine Tests

From the `backend` folder, run:

```bash
python test_engine.py
```

The test suite prints predicted category, risk score, expected result, and pass/fail for legitimate and fraudulent examples.

## Backend Setup

From the `backend` folder, run:

```bash
pip install -r requirements.txt
python train.py
python app.py
```

The backend runs at:

```text
http://localhost:5000
```

## Frontend Setup

From the `frontend` folder, run:

```bash
npm install
npm run dev
```

The frontend runs at:

```text
http://localhost:5173
```

To use a different backend URL, create a frontend environment variable:

```bash
VITE_API_BASE_URL=http://localhost:5000
```

## API

### Health Check

```http
GET /health
```

Response:

```json
{
  "status": "healthy"
}
```

### Predict Fraud Risk

```http
POST /predict
Content-Type: application/json
```

Request:

```json
{
  "text": "Your KYC will expire today. Verify immediately."
}
```

Response:

```json
{
  "risk_score": 94,
  "ml_score": 87,
  "rule_score": 100,
  "confidence": 0.94,
  "risk_level": "High",
  "category": "KYC Scam",
  "matched_patterns": [
    "KYC update or expiry language",
    "Identity verification request",
    "Urgent completion pressure"
  ],
  "keywords": ["kyc", "verify", "immediately"],
  "explanation": "Message contains KYC verification language and urgency indicators."
}
```

Actual scores can vary slightly based on the trained model and dataset split.

### Model Metrics

```http
GET /metrics
```

Response:

```json
{
  "accuracy": 0.91,
  "precision": 0.89,
  "recall": 0.93,
  "f1_score": 0.91,
  "confusion_matrix": [[4, 1], [0, 6]]
}
```

### Prediction History

Prediction history is stored only in the user's browser with `localStorage`.
The backend does not store user-submitted messages or prediction history.
The frontend keeps the latest 50 analyses and includes a Clear History button.

## Notes

- No database is required.
- CORS is enabled in `app.py`.
- If `/predict` is called before training, the API returns a clear error asking you to run `python train.py`.
- `train.py` creates `backend/model/model.pkl` and `backend/model/vectorizer.pkl`.
- `train.py` creates `backend/model/metrics.json`.
- Successful `/predict` calls are stored in browser `localStorage`, so each user only sees their own history.
